## 1. 제품 포지셔닝

### 한 줄 정의

> **"AI가 매일 읽어주는 기술 뉴스룸"** — 18개 글로벌 테크 소스를 30분마다 수집하고, Claude AI가 한국어로 요약·분석·점수화하여 개발자에게 큐레이션 해주는 플랫폼

### 핵심 가치 제안 (Value Proposition)

| 사용자 Pain Point | Mud의 해결책 |
|---|---|
| "매일 기술 뉴스를 챙기고 싶은데 소스가 너무 많다" | 18개 소스를 자동 수집, 하나의 대시보드로 통합 |
| "영어 아티클을 읽을 시간이 부족하다" | Claude AI가 한국어 요약을 자동 생성 |
| "중요한 것과 덜 중요한 것을 구분하기 어렵다" | AI 관련성 점수(1~5점)로 우선순위 제공 |
| "깊이 있는 분석이 필요할 때가 있다" | 원클릭 AI 심층분석 (최대 4,096 토큰) |
| "내 관심 분야만 골라보고 싶다" | 9개 카테고리, 소스별·점수별·키워드 필터링 |

### 타겟 페르소나

| 페르소나 | 설명 | 핵심 니즈 | 사용 시나리오 |
|---|---|---|---|
| **🧑‍💻 주니어 개발자** | 1~3년차, 기술 트렌드에 민감 | 업계 흐름 파악, 학습 방향 설정 | 출근 전/점심시간에 트렌드 브라우징 |
| **👨‍💼 테크 리드/CTO** | 기술 의사결정권자 | 기술 선택 근거, 팀 공유 자료 | 주간 기술 동향 리뷰, 팀 회의 자료 |
| **📚 취준생/전환자** | 개발 직군 준비 중 | 최신 기술 스택 파악, 면접 준비 | 매일 트렌드 확인으로 업계 감각 유지 |
| **🔬 AI/ML 연구자** | 논문·모델 추적 필요 | ArXiv, HuggingFace, Papers With Code 통합 모니터링 | 새 논문·모델 빠른 스캔 |

## 2. 경쟁 분석

### 직접 경쟁

| 서비스 | 특징 | Mud 대비 차별점 |
|---|---|---|
| **GeekNews** | 한국 개발자 커뮤니티, 사용자 투고 | Mud는 자동 크롤링 + AI 분석. 사람 의존 X |
| **TLDR Newsletter** | 이메일 뉴스레터 (영어) | Mud는 실시간 웹 대시보드 + 한국어 |
| **daily.dev** | 브라우저 확장, 영어 개발 뉴스 | Mud는 AI 분석·요약 + 한국어 특화 |
| **Hacker News** | 커뮤니티 투표 기반 랭킹 | Mud는 AI 점수 + 다중 소스 통합 |

### Mud의 방어 가능한 차별화

1. **한국어 AI 요약** — 영어 원문을 읽을 필요 없음 (국내 유일)
2. **18개 소스 통합** — 단일 소스가 아닌 종합 큐레이션
3. **AI 관련성 점수** — 사람 투표가 아닌 AI 기반 중요도 판단
4. **심층 분석 온디맨드** — 클릭 한 번으로 상세 AI 분석 생성
5. **30분 주기 갱신** — 뉴스레터(1일)보다 훨씬 빠른 갱신

### SWOT 분석

| | 긍정적 | 부정적 |
|---|---|---|
| **내부** | **강점(S)**: AI 분석 자동화, 다중 소스 통합, 한국어 특화, 깔끔한 다크모드 UI, 9개 카테고리 분류 | **약점(W)**: 모바일 미대응, SEO 부재, 테스트 코드 0건, 보안 미비, 브랜딩 에셋 없음 |
| **외부** | **기회(O)**: 한국 개발자 시장 성장, AI 뉴스 큐레이션 수요 급증, 한국어 기술 콘텐츠 부족 | **위협(T)**: 대형 서비스(Perplexity, ChatGPT)의 뉴스 요약 기능, 기존 개발자 커뮤니티(velog, GeekNews)의 네트워크 효과 |

## 3. 브랜딩 전략

### 현재 상태

- **이름**: Mud (⚗️)
- **태그라인**: "기술 트렌드 큐레이션"
- **로고**: 이모지 기반 (⚗️), 전용 로고 없음
- **컬러**: 다크 모드 (#0f1117 배경, #6366f1 인디고 액센트)

### 브랜딩 강화 제안

#### 네이밍 스토리

"Mud(진흙)"는 연금술에서 귀금속을 만들어내는 원재료를 상징합니다. **흩어진 정보(진흙)에서 가치 있는 인사이트(금)를 추출한다**는 의미를 담고 있습니다.

#### 슬로건 후보

| 안 | 슬로건 | 콘셉트 |
|---|---|---|
| A | **"기술의 흐름을, AI가 읽어드립니다"** | AI 자동화 + 편의성 강조 |
| B | **"18개 소스, 하나의 대시보드"** | 통합 큐레이션 강조 |
| C | **"매일 30분, 개발자의 아침 루틴"** | 습관화/라이프스타일 |
| D | **"정보의 홍수에서 핵심만"** | 필터링/큐레이션 가치 |

**권장: A안** — AI 기반 서비스의 핵심 차별점을 가장 직관적으로 전달

#### 비주얼 아이덴티티 투자 (Phase 2 연계)

| 에셋 | 용도 | 우선순위 |
|---|---|---|
| SVG 로고 (⚗️ → 커스텀) | 웹, OG 이미지, 파비콘 | P1 |
| OG 이미지 템플릿 | 소셜 공유 시 미리보기 | P1 (SEO 이슈 해결 시) |
| 컬러 팔레트 가이드 | 일관된 UI/마케팅 자료 | P2 |
| 이메일 템플릿 | 뉴스레터 (Phase 3) | P3 |

## 4. Go-To-Market (GTM) 전략

### Phase 1: 인지도 확보 (Sprint 1~2 완료 후, ~4월 중순)

> **목표**: 얼리어답터 개발자 500명 유입

Sprint 1의 보안/안정성 이슈가 해결된 후 본격 마케팅을 시작합니다.

#### 4.1.1 개발자 커뮤니티 침투

| 채널 | 전략 | 예상 효과 |
|---|---|---|
| **GeekNews** | "사이드 프로젝트 공유" 형태로 포스팅. AI 크롤링 기술 스택 설명 중심 | 초기 트래픽 유입, 피드백 수집 |
| **Reddit r/korea, r/programming** | 영문 포스트 "Built an AI-powered Korean tech trend aggregator" | 글로벌 개발자 관심 |
| **Hacker News** | "Show HN" 포스트 | 대규모 트래픽 가능성 (모바일 대응 필수) |
| **커리어리/블라인드** | "개발자 필수 북마크" 형태의 자연스러운 소개 | 한국 개발자 직접 도달 |

#### 4.1.2 기술 블로그 시리즈

> "Mud를 만들며 배운 것" 시리즈 — 제품 홍보와 기술 공유를 동시에

| 편 | 주제 | 플랫폼 |
|---|---|---|
| 1편 | "18개 테크 소스를 크롤링하는 아키텍처" | velog, medium |
| 2편 | "Claude AI로 기술 뉴스 자동 분석하기" | velog, medium |
| 3편 | "Next.js 14 + Spring Boot 풀스택 사이드 프로젝트" | velog |
| 4편 | "사이드 프로젝트를 Railway에 배포하기" | velog |

**핵심**: 각 글 말미에 자연스럽게 Mud 링크 삽입. "직접 써보세요 → mud-frontend-production.up.railway.app"

#### 4.1.3 GitHub 프로젝트 최적화

| 항목 | 현재 | 개선 |
|---|---|---|
| README.md | 없음 (CLAUDE.md만 존재) | 스크린샷 포함 README 작성 |
| GitHub Topics | 미설정 | `tech-news`, `ai-curation`, `korean`, `nextjs`, `spring-boot` |
| GitHub Description | 미설정 | "AI-powered Korean tech trend curation platform" |
| Demo URL | 미설정 | Railway 프로덕션 URL 연결 |
| License | 미설정 | MIT 또는 적절한 라이선스 추가 |

### Phase 2: 성장 가속 (Sprint 2~3 완료 후, ~5월)

> **목표**: MAU 2,000명, 일평균 방문 200+

모바일 대응(F5)과 SEO(F6)가 완료된 후 실행합니다.

#### 4.2.1 SEO 기반 오가닉 트래픽

Sprint 2의 SEO 이슈(F6) 해결 후 즉시 효과 발생:

| SEO 전략 | 상세 |
|---|---|
| **페이지별 메타데이터** | 트렌드 상세 페이지마다 고유 title, description, OG 이미지 |
| **sitemap.xml** | 트렌드 아이템별 URL 자동 생성 |
| **구조화 데이터** | Article, BreadcrumbList JSON-LD |
| **타겟 키워드** | "기술 트렌드", "개발 뉴스 한국어", "AI 기술 뉴스", "개발자 뉴스 큐레이션" |

검색 유입 시나리오: 사용자가 "GitHub 트렌딩 한국어"를 검색 → Mud의 GitHub Trending 트렌드 상세 페이지 노출

#### 4.2.2 소셜 미디어 자동화

| 채널 | 전략 | 구현 |
|---|---|---|
| **Twitter/X** | 일일 "오늘의 Top 5 기술 트렌드" 자동 포스팅 | 관련성 점수 Top 5 아이템 API → 자동 트윗 |
| **LinkedIn** | 주간 "이번 주 기술 동향" 카드 뉴스 | 수동 큐레이션 → Canva 카드 |
| **Telegram/Discord** | "Mud 트렌드 봇" — 새 고점수 트렌드 자동 알림 | Webhook 연동 |

#### 4.2.3 이메일 뉴스레터

> "Mud Weekly" — 주간 기술 트렌드 다이제스트

| 항목 | 상세 |
|---|---|
| 발송 주기 | 매주 월요일 오전 8시 |
| 콘텐츠 | 지난 주 관련성 점수 상위 10개 + AI 요약 |
| 구독 경로 | 사이트 내 이메일 구독 폼 |
| 도구 | Resend 또는 Mailchimp 무료 티어 |

### Phase 3: 리텐션 강화 + 수익화 (6월~)

> **목표**: 주간 리텐션 40%+, 수익 모델 검증

#### 4.3.1 사용자 참여 기능 (개발 로드맵 Phase 3 연계)

| 기능 | 마케팅 효과 |
|---|---|
| **OAuth 로그인** (GitHub/Google) | 가입 허들 최소화, 사용자 데이터 확보 |
| **서버사이드 북마크** | 디바이스 간 동기화 → 앱 사용 습관 강화 |
| **키워드 구독 알림** | "내가 관심있는 기술만" → 일일 재방문 유도 |
| **주간 리포트** | 이메일 뉴스레터 자동 생성 → 리텐션 |
| **RSS 피드 제공** | 외부 채널 유통 → 신규 유입 |

#### 4.3.2 수익화 후보 모델

| 모델 | 설명 | 적합성 |
|---|---|---|
| **프리미엄 구독** | 심층분석 무제한, 고급 필터, API 접근 | ⭐⭐⭐ — 핵심 가치와 직결 |
| **기업 대시보드** | 팀용 트렌드 모니터링, 맞춤 소스 설정 | ⭐⭐⭐ — B2B 확장 |
| **스폰서 콘텐츠** | 기술 기업 스폰서 트렌드 하이라이트 | ⭐⭐ — 트래픽 확보 후 |
| **API 유료화** | 트렌드 데이터 API 외부 판매 | ⭐⭐ — 데이터 축적 후 |
| **광고** | 배너/네이티브 광고 | ⭐ — UX 저해, 최후 수단 |

**권장: 프리미엄 구독 모델**
- Free: 일일 트렌드 조회 + 3회 심층분석
- Pro (₩9,900/월): 무제한 심층분석 + 키워드 알림 + API 접근 + 주간 리포트

## 7. 마케팅 전 필수 선행 작업

### 🔴 마케팅 시작 전 반드시 해결해야 할 이슈

마케팅으로 유입된 사용자가 나쁜 첫인상을 받으면 재방문 확률이 급감합니다. 아래 이슈는 **마케팅 시작 전 필수 해결** 항목입니다:

| 이슈 | 이유 | 스프린트 |
|---|---|---|
| **Admin API 보안** | 악의적 사용자가 전체 캐시 삭제/비용 발생 가능 | Sprint 1 |

## 9. 핵심 정리

### 즉시 실행 가능한 3가지

1. **GitHub README 작성** — 스크린샷 포함, 프로젝트 소개 최적화 (공수 0.5일)
2. **기술 블로그 1편 집필 시작** — "18개 소스를 크롤링하는 아키텍처" (공수 1일)
3. **Google Analytics 설치** — 현재 트래픽 베이스라인 측정 (공수 0.25일)

### 마케팅 타이밍 핵심

```
❌ 지금 마케팅 시작 → 보안 취약, 모바일 깨짐, SEO 없음 → 첫인상 망침
✅ Sprint 2 완료 후 (4/18~) → 안정적 서비스 + 모바일 대응 + SEO → 확산 기반 확보
```

### 가장 임팩트 높은 마케팅 채널 (우선순위)

1. **GeekNews** — 한국 개발자 직접 도달, 피드백 루프 빠름
2. **기술 블로그 시리즈** — SEO 장기 효과 + 기술력 어필
3. **GitHub 프로젝트 최적화** — 오픈소스 개발자 유입의 관문
4. **Hacker News Show HN** — 글로벌 노출, 대량 트래픽 가능
5. **이메일 뉴스레터** — 리텐션 최고 채널, 구독자 자산화
